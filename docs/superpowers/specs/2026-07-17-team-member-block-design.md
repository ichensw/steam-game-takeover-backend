# 队伍成员拉黑准入设计

## 目标

当用户申请加入接龙时，除接龙创建者外，也检查当前已加入成员是否拉黑该用户。命中任意一人则拒绝加入，但不向申请人透露具体限制者。

## 规则

- 拉黑关系保持单向：A 拉黑 B，仅影响 B 加入含有 A 的队伍。
- 仅 `member_state = joined` 的成员参与校验；A 退出或被踢出后，不再影响该队伍的后续加入。
- 创建者的现有拉黑校验继续保留，即使创建者未作为成员记录存在也生效。
- 不改变队伍浏览、成员展示、退出队伍或踢人逻辑。
- 加入被拒绝时统一返回：`当前接龙暂时无法加入，请稍后重试。`
- 前端沿用接口的通用失败提示，不展示“拉黑”、限制者身份或对应错误码。

## 实现

- 在加入接龙的同一事务内，用 `ttw_takeover_member` 与 `ttw_user_block` 查询是否存在已加入成员拉黑申请人。
- 命中时复用加入失败分支，返回不暴露原因的通用响应。
- 为“成员拉黑拒绝加入”“成员退出后允许加入”“非成员拉黑不影响加入”增加后端测试。

## 边界

- 不做双向屏蔽：B 拉黑 A 不阻止 B 加入含有 A 的队伍。
- 不追溯历史成员；只有当下队伍成员生效。
